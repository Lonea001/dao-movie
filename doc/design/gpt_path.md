## 为什么这样定

因为你不是在做商用系统，你是在做毕业设计。
毕业设计老师通常更看重：

* 题目和实现是否一致
* 页面和功能是否完整
* 数据库和系统结构是否清楚
* 论文是否写得像样
* 你能不能讲清楚

不是看你有没有堆很多高级中间件。

## 数据来源就这样写

你已经说了：**网上找 1000 条热门电影数据就够了。**

论文里建议写成：

> 本系统的数据主要来源于公开网络影视信息，选取约 1000 条热门电影基础数据作为实验与展示数据。为保证系统演示效果，对原始数据进行了清洗、去重、字段统一和格式规范化处理，最终导入 MySQL 数据库用于网站展示、分类筛选及关键词检索。

这样写就行。
不要在论文里把重点放在“爬虫技术”上，除非你真的会做。

## 开题报告里应该怎么写

你现在最需要的是让开题内容和最终代码一致。
所以开题不要再写注册、评论、多用户那些东西。

### 课题意义

可以写成：

> 随着互联网技术和数字媒体的发展，影视信息获取方式逐渐网络化、便捷化。为了更高效地展示电影基础信息、热门排行和分类内容，设计并实现一个影视数据展示网站具有一定的现实意义。该系统能够帮助用户快速浏览影视数据、查询影片信息，并通过点赞和收藏功能增强网站的实用性与交互性。同时，本课题也能加深对前后端分离开发模式、数据库设计和 Web 系统实现过程的理解。

### 研究内容

可以写成：

> 本课题拟设计并实现一个影视数据展示网站，主要包括首页展示、影视列表、分类浏览、详情查看、关键词搜索、排行榜、点赞和收藏等功能。系统以前后端分离方式开发，前端使用 Vue3 实现页面展示与交互，后端使用 Spring Boot 提供接口服务，MySQL 用于存储影视数据及点赞收藏数据。通过导入约 1000 条公开电影数据，完成系统的数据展示、查询及管理功能。

### 技术路线

可以写成：

> 系统采用前后端分离架构。前端基于 Vue3、Vue Router 和 Axios 实现页面构建与接口调用；后端基于 Spring Boot 和 MyBatis 实现业务逻辑与数据访问；数据库采用 MySQL 存储电影信息及用户行为数据。系统通过关键词模糊查询实现影视搜索，通过分页与分类筛选实现数据浏览，通过点赞和收藏表实现管理员对影片的个性化标记功能。

## 你这个项目的“创新点”不要写太大

别写推荐算法、智能分析这些你做不到的。
就写这种很稳的：

1. 采用前后端分离方式实现影视数据展示网站，提高系统结构清晰度。
2. 实现电影数据的分类展示、关键词搜索与排行榜浏览，提升信息获取效率。
3. 通过点赞与收藏功能增强网站交互性和个性化展示效果。

这类创新点虽然不强，但**真实**，老师更容易接受。
